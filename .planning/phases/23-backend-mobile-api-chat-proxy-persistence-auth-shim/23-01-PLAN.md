Module docstring states: "SPIKE D-31 (Wave 0 BLOCKING) — Configure GZipMiddleware(minimum_size=1024) on a tiny FastAPI app, fire a streaming/SSE response through it, assert chunks are delivered as emitted (un-buffered), NOT batched at end-of-stream. PASS criterion: time-to-first-byte for chunk N ≤ 200ms after server emits chunk N. FAIL → switch to content-type-exclude config explicitly listing text/event-stream and re-spike. Per Starlette ≥0.46.0 release notes, text/event-stream is in DEFAULT_EXCLUDED_CONTENT_TYPES — this spike is regression-prevention only."

Implementation strategy (the simplest path that proves un-buffered behavior — do NOT require booting the full api_server app):

1. Construct a minimal FastAPI app inline in the test:
   ```python
   from fastapi import FastAPI
   from fastapi.responses import StreamingResponse
   from starlette.middleware.gzip import GZipMiddleware

   app = FastAPI()
   app.add_middleware(GZipMiddleware, minimum_size=1024)

   @app.get("/sse")
   async def sse():
       async def gen():
           # emit 5 chunks 100ms apart, each padded so total > 1024 bytes
           import asyncio
           for i in range(5):
               yield f"data: chunk-{i}\n\n".encode() + (b"x" * 300) + b"\n"
               await asyncio.sleep(0.1)
       return StreamingResponse(gen(), media_type="text/event-stream")
   ```

2. Drive it with httpx.AsyncClient(transport=httpx.ASGITransport(app)) and a streaming GET (`async with client.stream("GET", "/sse") as r:`), record arrival time of each chunk via `time.monotonic()` deltas.

3. Assert the first chunk arrives within 250ms of the request start (i.e. NOT batched at end-of-stream which would be ~500ms+).

4. Assert `r.headers.get("content-encoding")` is NOT "gzip" for the SSE response (i.e. starlette excluded it). If it IS gzip, fail with a clear message: "REGRESSION: text/event-stream IS being compressed. Switch to explicit content-type-exclude config."

5. Add a SECOND test in the same file that verifies a NON-SSE response (e.g. `@app.get("/json")` returning 2KB JSON via JSONResponse) DOES get gzipped when the client sends `Accept-Encoding: gzip` — proves GZipMiddleware is actually engaged.

Marker: `@pytest.mark.asyncio` (NOT `pytest.mark.api_integration` — this is a spike, runs without testcontainers per `tests/spikes/test_respx_authlib.py:23` precedent).

Test functions:
- `async def test_sse_chunks_arrive_unbatched_under_gzip_middleware():`
- `async def test_non_sse_response_is_actually_gzipped():`

This spike is INDEPENDENT of the rest of api_server — it does NOT import the production main.py app. It constructs its own app inline so the spike survives even if main.py changes.
