<objective>
Build the **reaper** — a 15s-tick asyncio task that sweeps `inapp_messages` rows stuck in `status='forwarded'` past the D-40 threshold (last_attempt_at < NOW() - 11 minutes). The reaper exists for genuine stuck rows where the api_server crashed mid-forward (between status='forwarded' UPDATE and the bot HTTP call's resolution), NOT for normal long inferences (which are bounded by the 600s httpx timeout in the dispatcher).

Per D-40 revision (no auto-retry): every stuck row is transitioned DIRECTLY to `status='failed'` with `last_error='reaper_timeout'`. The reaper does NOT requeue; clients retry by sending a new message.

Per D-32: `FOR UPDATE SKIP LOCKED` ensures multi-replica safety; the reaper never collides with the dispatcher's in-flight rows.

Purpose: Close the durability gap when api_server crashes mid-forward AND restart_sweep didn't catch the row (e.g., crash happened < 15 minutes before restart, so restart_sweep's threshold didn't trigger; or container was restarted without restart_sweep firing for some reason).
Output: reaper service + 5+ integration tests.
</objective>

```python
"""Phase 22c.3 — inapp_messages stuck-forwarded reaper.

Lifespan-managed asyncio task. Ticks every 15s; selects rows in
status='forwarded' with last_attempt_at older than 11 minutes (D-30 +
D-40 revision: 10min bot timeout + 1min slack). Per D-40 (no auto-retry),
every stuck row is transitioned directly to 'failed' with
last_error='reaper_timeout'. The reaper does NOT requeue.

Why this exists: api_server crashes between status='forwarded' UPDATE
and the dispatcher's status='done|failed' UPDATE leave a row stuck. The
restart sweep (D-31, 15min threshold) catches MOST of these, but if the
crash happens close to a restart (or restart_sweep itself fails), the
reaper is the steady-state safety net.

Multi-replica safety (D-32): FOR UPDATE SKIP LOCKED in the store query
guarantees no two reaper ticks (or a reaper + a dispatcher) can pick the
same row.
"""
from __future__ import annotations
import asyncio
import logging

from . import inapp_dispatcher  # for _mark_failed
from . import inapp_messages_store as ims

_log = logging.getLogger("api_server.inapp_reaper")

REAPER_TICK_S = 15.0
REAPER_BATCH_LIMIT = 50
STUCK_THRESHOLD_MINUTES = 11   # D-40: 10min bot timeout + 1min slack


async def reaper_loop(app_state, stop_event: asyncio.Event) -> None:
    """Lifespan-managed reaper. Runs until stop_event is set."""
    while not stop_event.is_set():
        try:
            await _sweep_once(app_state)
        except Exception:
            _log.exception("phase22c3.reaper.sweep_failed")
        try:
            await asyncio.wait_for(stop_event.wait(), REAPER_TICK_S)
        except asyncio.TimeoutError:
            pass


async def _sweep_once(app_state) -> int:
    """Pick stuck rows + transition each to failed. Returns count."""
    pool = app_state.db
    # Scope-1: pick stuck rows (FOR UPDATE SKIP LOCKED). The store function
    # holds the FOR UPDATE lock for the duration of this acquire().
    async with pool.acquire() as conn:
        async with conn.transaction():
            stuck = await ims.fetch_stuck_forwarded(
                conn, STUCK_THRESHOLD_MINUTES, REAPER_BATCH_LIMIT,
            )
            if not stuck:
                return 0
            # Hold the FOR UPDATE lock briefly: do the transition INSIDE
            # this same tx so SKIP LOCKED protects us from a parallel
            # reaper picking the same rows. This means we DON'T release
            # the connection across the agent_events INSERT — but
            # agent_events INSERT is fast (no httpx call), so Pitfall 4
            # doesn't apply here.
            #
            # We need the agent_container_id (= container_row_id) to
            # write the agent_events row. fetch_stuck_forwarded only
            # returns inapp_messages columns, so we re-query for the
            # container_row_id by JOINing through agent_id.
            for row in stuck:
                # Get container_row_id for the agent_events INSERT
                container_row_id = await conn.fetchval(
                    """
                    SELECT id FROM agent_containers
                    WHERE agent_instance_id=$1
                    ORDER BY stopped_at DESC NULLS LAST, created_at DESC
                    LIMIT 1
                    """,
                    row["agent_id"],
                )
                if container_row_id is None:
                    # No container for this agent — just mark failed
                    # without an event (caller can detect via status).
                    await ims.mark_failed(conn, row["id"], "reaper_timeout")
                    continue
                # mark_failed + insert event in same tx
                await ims.mark_failed(conn, row["id"], "reaper_timeout")
                from .event_store import insert_agent_event
                from datetime import datetime, timezone
                await insert_agent_event(
                    conn,
                    container_row_id,
                    "inapp_outbound_failed",
                    {
                        "error_type": "reaper_timeout",
                        "message": "reaper_timeout",
                        "retry_count": int(row["attempts"]),
                        "captured_at": datetime.now(timezone.utc).isoformat(),
                    },
                    correlation_id=str(row["id"]),
                    published=False,
                )
            return len(stuck)
```

```python
async def test_reaper_loop_responds_to_stop_event(db_pool):
```
