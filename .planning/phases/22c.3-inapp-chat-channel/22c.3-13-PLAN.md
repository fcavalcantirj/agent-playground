<objective>
Substitute picoclaw with zeroclaw in the Phase 22c.3 5-recipe matrix per user direction (2026-04-30, "ditch picoclaw — DEFER. leave for last. pick the most prominent one"). Per the project's stars-desc backlog (`recipes/BACKLOG.md`), ZeroClaw is top-of-queue at 30,845 ★ (Rust, distroless, ~50 MB image at `ghcr.io/zeroclaw-labs/zeroclaw:latest`).

This plan creates `recipes/zeroclaw.yaml` (NEW file) that declares a v0.2 recipe with a fully-formed `channels.inapp` block. ZeroClaw natively exposes `POST /webhook` returning `{"response":"<reply>","model":"<model>"}` synchronously — no sidecar, no bridge, no subprocess fork. Idempotency is built-in via `X-Idempotency-Key`. Multi-turn via `X-Session-Id`. Headless mode via `gateway.require-pairing=false`. All four were empirically PROVED 2026-04-30 with real OpenRouter LLM in 3-second round-trip (full evidence: `spikes/recipe-zeroclaw.md`).

The recipe's `build.mode: image_pull` (per RECIPE-SCHEMA.md §3.2) avoids a 10-minute Rust cold-build by pulling the prebuilt distroless image from ghcr.

**picoclaw is NOT modified by this plan.** `recipes/picoclaw.yaml` stays in repo for backward compatibility with the smoke suite, but is OUT of Phase 22c.3 inapp scope. Reintegrating picoclaw later is a separate phase.

Per D-22 dumb-pipe: api_server's dispatcher POSTs `{"message":"<user_text>"}` and reads `body["response"]`. Bot owns its memory via the `/zeroclaw-data/.zeroclaw/` SQLite session store + auto-save. No api-side prompt composition.

Per D-20: ZeroClaw's gateway IS the inapp surface; channels.telegram et al. are additional channels sharing the same agent loop. v1 is single-channel-per-instance.

Purpose: Fifth of 5 recipes — picoclaw substitute, simplest of the matrix (no sidecar, no config-flag gymnastics, just pull + onboard + 3× config set + daemon).
Output: `recipes/zeroclaw.yaml` parses; api_server's `load_all_recipes(...)` at boot picks it up; recipe is selectable via `GET /v1/recipes` once the api_server image rebuilds.

**From spikes/recipe-zeroclaw.md (verbatim) — verified boot + chat sequence:**

```bash
# 1. Pull prebuilt image (distroless, ~50 MB)
docker pull ghcr.io/zeroclaw-labs/zeroclaw:latest

# 2. Onboard — writes /zeroclaw-data/.zeroclaw/config.toml
docker run --rm -v zeroclaw-data:/zeroclaw-data \
  --entrypoint zeroclaw ghcr.io/zeroclaw-labs/zeroclaw:latest \
  onboard --quick --force \
  --provider openrouter --api-key "$OPENROUTER_API_KEY" \
  --model anthropic/claude-haiku-4.5

# 3. Permit non-loopback bind
docker run --rm -v zeroclaw-data:/zeroclaw-data \
  --entrypoint zeroclaw ghcr.io/zeroclaw-labs/zeroclaw:latest \
  config set gateway.allow-public-bind true

# 4. Bind 0.0.0.0
docker run --rm -v zeroclaw-data:/zeroclaw-data \
  --entrypoint zeroclaw ghcr.io/zeroclaw-labs/zeroclaw:latest \
  config set gateway.host '0.0.0.0'

# 5. Disable pairing (headless dispatcher use)
docker run --rm -v zeroclaw-data:/zeroclaw-data \
  --entrypoint zeroclaw ghcr.io/zeroclaw-labs/zeroclaw:latest \
  config set gateway.require-pairing false

# 6. Run daemon
docker run -d -p 42617:42617 -v zeroclaw-data:/zeroclaw-data \
  -e OPENROUTER_API_KEY="$OPENROUTER_API_KEY" \
  --entrypoint zeroclaw ghcr.io/zeroclaw-labs/zeroclaw:latest \
  daemon

# 7. Chat round-trip (sync, ~3s)
curl -X POST http://localhost:42617/webhook \
  -H 'Content-Type: application/json' \
  -d '{"message":"who are you in 1 short sentence?"}'
# → HTTP 200
# {"model":"anthropic/claude-haiku-4.5","response":"I'm ZeroClaw, ..."}
```

**channels.inapp YAML schema** (this plan's authoritative shape):

```yaml
channels:
  inapp:
    config_transport: pre_start_only
    transport: http_localhost
    port: 42617
    contract: zeroclaw_native        # NOT openai_compat — {message} → {response, model}
    endpoint: /webhook
    method: POST
    auth_mode: none                  # gateway.require-pairing=false; loopback-trusted
    idempotency_header: X-Idempotency-Key
    session_header: X-Session-Id
    request_envelope:
      message: "{prompt}"
    response_envelope:
      reply_path: "$.response"
      model_path: "$.model"
    streaming:
      transport: ws
      path: /ws/chat
      message_envelope:
        client_send: { type: "message", content: "{prompt}" }
        server_chunk: { type: "chunk", content: "<text>" }
        server_done: { type: "done", full_response: "<text>" }
    health_endpoint: /health
    ready_log_regex: "ZeroClaw Gateway listening on"
    metrics_endpoint: /metrics
    persistent_argv_override:
      entrypoint: zeroclaw
      argv: [daemon]
      pre_start_commands:
        - argv: [zeroclaw, onboard, --quick, --force, --provider, openrouter, --api-key, "${OPENROUTER_API_KEY}", --model, "$MODEL"]
        - argv: [zeroclaw, config, set, gateway.allow-public-bind, "true"]
        - argv: [zeroclaw, config, set, gateway.host, "0.0.0.0"]
        - argv: [zeroclaw, config, set, gateway.require-pairing, "false"]
    activation_env:
      OPENROUTER_API_KEY: "${INAPP_PROVIDER_KEY}"
      ZEROCLAW_WORKSPACE: "/zeroclaw-data/workspace"
```
