## Decisions Made

- **dispatcher_loop signature stays single-arg.** The plan's example used `dispatcher_loop(app.state, app.state.inapp_stop)` but the source-of-truth file in services/inapp_dispatcher.py defines `dispatcher_loop(state) -> None` and reads `getattr(state, "inapp_stop", None)` internally. Lifespan respects the existing contract: it wires `state.inapp_stop` onto state and passes only `state`. The 2 OTHER loops (reaper, outbox) accept `(state, stop_event)` positionally per their actual signatures.
- **Lifespan ordering: inapp BEFORE 22b watcher re-attach.** The plan said "after line ~91" which would have placed inapp wiring after 22b. I moved it BEFORE because the 22b block calls `docker.from_env()` + `containers.get(...)` — Docker can hang on a dead daemon, and a fail-loud Redis ping should NOT be gated on Docker responding first. If Redis is unreachable the boot should die fast, not wait on Docker timeouts. Documented inline in the ordering rationale comment.
- **5s drain budget.** Plan said 5s aggregate. Empirical live test showed both worker processes drain within milliseconds (no `Application shutdown complete` log line was preceded by a wait). Budget is 5s for safety, not because we expect to need it.
- **conftest.async_client now depends on redis_container.** The lifespan ping is fail-loud — every test that uses `async_client` would have failed without an environment-injected Redis. Two reasonable solutions: (a) make Redis optional in the lifespan (rejected — defeats the fail-loud invariant the plan REQUIRES), or (b) wire the testcontainer Redis URL into `async_client`. Picked (b). Now the test surface matches prod surface.
- **No structlog bridge for stdlib `_log.info(...)` lines.** The lifespan emits `_log.info("phase22c3.lifespan.inapp_tasks_started", ...)` lines via `logging.getLogger("api_server.main")`, but the running container does NOT show them in `docker logs` output — the stdlib root logger has no handler configured (configure_logging only sets up structlog). This is a pre-existing gap (`phase22b.reattach.*` logs also don't surface). Out of scope for this plan; recommend a future "stdlib logging bridge to structlog" plan to fix all `phase*.*.*` lines uniformly. Smoke test verified the lifespan ran correctly via behavior (PG → Redis publish, restart_sweep effect, clean shutdown), not log lines.
