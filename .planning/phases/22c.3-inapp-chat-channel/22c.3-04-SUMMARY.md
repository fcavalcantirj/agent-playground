## Decisions Made

- **dispatcher's readiness gate stays in the dispatcher (D-37) — NOT in the store.** `fetch_pending_for_dispatch` returns ALL pending rows joined to their container, regardless of `container_status`. The dispatcher (Plan 22c.3-05) decides whether to forward, mark `container_not_ready` failed, or leave pending. Test `test_fetch_pending_joins_agent_containers` verifies a `starting` container still surfaces.
- **`fetch_history_for_agent` ordering: `ORDER BY created_at DESC`.** This powers the future REST history endpoint (Plan 22c.3-08) which renders newest-first. SSE replay queries from `agent_events` separately, ascending by `seq`, so the two reads do not conflict.
- **`delete_history_for_agent_user` does NOT cascade to `agent_events`.** `agent_events` is keyed by `agent_container_id`, not `agent_id`. The route handler issues a sibling DELETE in the same transaction; the store is single-purpose.
- **`mark_forwarded` empty-list guard.** A drained queue legitimately yields an empty `fetch_pending_for_dispatch` result; the bulk UPDATE shortcut returns early to avoid a round-trip with `WHERE id = ANY('{}'::uuid[])`.
- **`asyncpg.execute` returns a command tag string ("UPDATE N", "DELETE N").** `restart_sweep` and `delete_history_for_agent_user` parse the trailing integer with a defensive `try/except` fallback to 0 so a malformed tag does not raise mid-sweep.
