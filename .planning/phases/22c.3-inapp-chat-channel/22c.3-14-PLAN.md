```yaml
channels:
  inapp:
    transport: http_localhost
    port: 3000
    contract: a2a_jsonrpc
    endpoint: /a2a
    method: POST
    auth_mode: none
    request_envelope:
      jsonrpc: "2.0"
      id: "{message_id}"
      method: "message/send"
      params:
        message:
          role: "user"
          parts:
            - kind: "text"
              text: "{prompt}"
          messageId: "{message_id}"
    response_envelope:
      reply_path: "$.result.artifacts[0].parts[0].text"
      task_id_path: "$.result.id"
      state_path: "$.result.status.state"
    streaming:
      transport: sse
      method: "message/stream"
      reply_path: "$.result.artifact.parts[0].text"
      delta_field: "append"
      done_field: "lastChunk"
    health_endpoint: /health
    ready_log_regex: "Gateway listening on"
    persistent_argv_override:
      entrypoint: sh
      argv:
        - "-c"
        - |
          set -e
          cat > /nullclaw-data/config.json <<EOF
          {
            "agents":{"defaults":{"model":{"primary":"openrouter/$MODEL"}}},
            "models":{"providers":{"openrouter":{"api_key":"${OPENROUTER_API_KEY}","base_url":"https://openrouter.ai/api/v1"}}},
            "gateway":{"port":3000,"host":"0.0.0.0","allow_public_bind":true,"require_pairing":false},
            "a2a":{"enabled":true,"name":"${INAPP_AGENT_NAME}","description":"Phase 22c.3 inapp","url":"${INAPP_AGENT_URL}","version":"0.3.0"}
          }
          EOF
          chown -R nobody:nobody /nullclaw-data
          exec nullclaw gateway --host 0.0.0.0 --port 3000
    activation_env:
      OPENROUTER_API_KEY: "${INAPP_PROVIDER_KEY}"
      INAPP_AGENT_NAME: "{agent_name}"
      INAPP_AGENT_URL: "{agent_url}"
```

Notes for the executor:
- `${OPENROUTER_API_KEY}`, `${INAPP_AGENT_NAME}`, `${INAPP_AGENT_URL}`, `$MODEL` are SHELL-expanded inside the container at exec time (they come from `activation_env` + the per-session env-injection in 22c.3-09); they are NOT YAML template placeholders.
- `{message_id}`, `{prompt}`, `{agent_name}`, `{agent_url}` ARE recipe-template placeholders the dispatcher substitutes at dispatch time.
- `gateway.require_pairing=false` is intentional + safe because the dispatcher reaches port 3000 only over the per-session Docker bridge (Plan 22c.3-09), never the public internet.
- DO NOT add an `apk add python3` step. DO NOT spawn a sidecar process. DO NOT introduce INAPP_BOT_BIN. The native nullclaw gateway IS the bridge.

Preserve everything else in recipes/nullclaw.yaml — the verified_cells the recipe already has for its non-inapp chat workflow, the install/build steps, the metadata. This plan is purely additive.
