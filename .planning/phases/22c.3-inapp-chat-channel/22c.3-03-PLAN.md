**File 1: `deploy/docker-compose.prod.yml`** — make TWO additive edits:

(a) Add a `redis` service block AFTER the `postgres` service, BEFORE `api_server`:
```yaml
  redis:
    image: redis:7-alpine
    restart: always
    # Loopback-only inside the compose network. NO `ports:` block —
    # external access blocked by default (D-08). Caddy is in front for
    # public ingress, not Redis.
    command: ["redis-server", "--protected-mode", "yes"]
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 3s
      retries: 5
```
(Note: do NOT add `--bind 127.0.0.1` — inside the compose network, the redis container's `redis` hostname resolves to its bridge-net IP; binding to 127.0.0.1 would prevent api_server from reaching it. compose's default bridge network IS the security boundary; absence of `ports:` keeps it loopback-only from the host perspective.)

(b) Edit the `api_server` service: add `AP_REDIS_URL` to environment, add `redis: condition: service_healthy` to depends_on:
```yaml
    environment:
      AP_ENV: prod
      AP_TRUSTED_PROXY: "true"
      AP_MAX_CONCURRENT_RUNS: "2"
      AP_RECIPES_DIR: /app/recipes
      AP_REDIS_URL: "redis://redis:6379/0"   # NEW (Phase 22c.3)
      DATABASE_URL: ...
      AP_CHANNEL_MASTER_KEY: ${AP_CHANNEL_MASTER_KEY}
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy   # NEW (Phase 22c.3)
```
