## Pre-discussion empirical work

User signal: *"dissecate msv and understand. and, you ARE allowed to make tests b4 comitting to the plan for all agents."*

Two empirical artifacts were generated before the gray areas were even narrowed:

### MSV deep recon (subagent ae6a633d20205f9f4, ~83s)

Findings cited inline in CONTEXT.md `<canonical_refs>`. Key takeaways:
- Inbound IPC: HTTP POST to `localhost:{port}/v1/chat/completions` (OpenAI-compat envelope)
- Outbound: Redis Pub/Sub channel `sse:notifications:<user_id>` → SSE handler with 30s heartbeat
- Persistence: single `user_messages` table with `direction` discriminator
- Auth: web POST has Bearer middleware; SSE has zero auth (we improve)
- Workflow orchestration: Temporal SendMessageWorkflow with 8 activities (we simplify to direct asyncio)

### FIFO bind-mount empirical spike (negative result)

Tested `mkfifo /tmp/spike-fifo + docker run -v /tmp/spike-fifo:/inbox alpine`. Container did NOT receive host-written data. Confirmed: FIFO inode read/write semantics don't propagate across Docker Desktop's gRPC-FUSE/virtiofs boundary on macOS. **STRIKES FIFO from the inbound transport menu.**

### Error event taxonomy on SSE

**Locked as D-24.** New `agent_events.kind` value `inapp_outbound_failed` with payload `{error_type, message, retry_count}`. Existing `agent_error` kind stays for non-chat errors (no conflation).

### SSE reconnection / replay protocol

**Locked as D-25 + D-26.** Standard SSE `Last-Event-Id` header pattern; replay from `agent_events.seq > N`. Bounded at 500 events to prevent memory blow-up; over-budget triggers `event: replay_truncated` so client falls back to history endpoint.
