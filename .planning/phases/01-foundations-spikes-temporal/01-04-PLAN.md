       ```yaml
             - POSTGRES_SEEDS=postgresql

             timeout: 5s
             retries: 5

       volumes:

       ```

       **Key design decisions reflected:**
       - Postgres hosts BOTH app DB and Temporal's DB (same instance in dev, per D-07)
       - All ports bound to 127.0.0.1 (never 0.0.0.0)
       - Temporal depends on postgresql with `condition: service_healthy` (Pitfall 1 from research)
       - Temporal UI on port 8233 (D-08), maps from container's 8080
       - `restart: on-failure` on Temporal to handle rare race conditions
       - `DB=postgres12` is the Temporal-internal driver name for Postgres

    1. Create `deploy/hetzner/bootstrap.sh`:
       ```bash
       #!/bin/bash
       set -euo pipefail
       SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
       echo "=== Agent Playground Host Bootstrap ==="
       echo "Date: $(date -u)"
       echo ""
       "$SCRIPT_DIR/install-docker.sh"
       "$SCRIPT_DIR/install-postgres.sh"
       "$SCRIPT_DIR/install-redis.sh"
       "$SCRIPT_DIR/install-temporal.sh"
       "$SCRIPT_DIR/harden-ufw.sh"
       echo ""
       echo "=== Bootstrap complete ==="
       echo "Next steps:"
       echo "  1. Copy .env to /opt/agent-playground/.env"
       echo "  2. Start Temporal: cd /opt/agent-playground && docker compose up -d"
       echo "  3. Start API: systemctl start agent-playground-api"
       ```

    2. Create `deploy/hetzner/install-docker.sh`:
       - Check if Docker is installed: `command -v docker`
       - If not: Install Docker CE from official apt repo (docker.com/linux/debian or ubuntu)
       - Configure `/etc/docker/daemon.json` (idempotent -- check if already configured):
         ```json
         {
           "userns-remap": "default",
           "log-driver": "json-file",
           "log-opts": {
             "max-size": "10m",
             "max-file": "3"
           }
         }
         ```
       - Restart Docker if daemon.json was modified
       - Verify: `docker info | grep -i "user namespace"` shows remapping active
       - Verify: `docker --version` shows 27.x

    3. Create `deploy/hetzner/install-postgres.sh`:
       - Check if Postgres 17 is installed: `pg_lsclusters | grep 17`
       - If not: Add PGDG apt repo, install `postgresql-17`
       - Configure `postgresql.conf`: `listen_addresses = 'localhost'` (idempotent check)
       - Create databases if not exist:
         - `agent_playground` (app database)
         - `temporal` (Temporal persistence -- auto-setup also creates this, but having it pre-created is cleaner)
       - Create users if not exist:
         - `ap_api` with password from env or prompt
         - `temporal` with password from env or prompt
       - Grant privileges: `ap_api` owns `agent_playground`, `temporal` owns `temporal`
       - Enable + start: `systemctl enable --now postgresql`
       - Verify: `pg_isready -h 127.0.0.1`

    4. Create `deploy/hetzner/install-redis.sh`:
       - Check if Redis is installed: `command -v redis-server`
       - If not: Add Redis official apt repo, install `redis-server`
       - Configure `/etc/redis/redis.conf` (idempotent):
         - `bind 127.0.0.1 ::1`
         - `maxmemory 256mb`
         - `maxmemory-policy allkeys-lru`
       - Enable + start: `systemctl enable --now redis-server`
       - Verify: `redis-cli -h 127.0.0.1 ping` returns PONG

    5. Create `deploy/hetzner/install-temporal.sh`:
       - Check if Docker is running (dependency)
       - Create `/opt/agent-playground/` directory if not exists
       - Copy or symlink `docker-compose.yml` from repo to `/opt/agent-playground/`
       - Create `.env` file with Temporal PG password (prompt if not set in env)
       - Start Temporal via docker compose: `cd /opt/agent-playground && docker compose up -d`
       - Wait for Temporal to be ready: retry loop checking `temporal operator namespace list` (max 30 attempts, 2s between)
       - Verify: `temporal operator namespace list` succeeds

    6. Create `deploy/hetzner/harden-ufw.sh`:
       - Check if UFW is installed: `command -v ufw`
       - If not: `apt-get install -y ufw`
       - Configure rules (idempotent -- ufw handles duplicates):
         - `ufw default deny incoming`
         - `ufw default allow outgoing`
         - `ufw allow ssh` (port 22)
         - `ufw allow 443/tcp` (HTTPS only)
       - Enable: `ufw --force enable`
       - Verify: `ufw status` shows only SSH and 443/tcp allowed
       - Note: Postgres (5432), Redis (6379), Temporal (7233, 8233) are NOT exposed -- loopback only

    7. Make all scripts executable: `chmod +x deploy/hetzner/*.sh`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| internet -> host | UFW is the perimeter; only 22/tcp and 443/tcp should be reachable |
| host -> containers | Docker daemon with userns-remap provides UID isolation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-13 | Information Disclosure | Postgres exposed to internet | mitigate | install-postgres.sh sets `listen_addresses = 'localhost'`; harden-ufw.sh blocks port 5432 from outside |
| T-1-14 | Information Disclosure | Redis exposed to internet | mitigate | install-redis.sh sets `bind 127.0.0.1 ::1`; harden-ufw.sh blocks port 6379 |
| T-1-15 | Elevation of Privilege | Container escaping via UID 0 | mitigate | install-docker.sh enables `userns-remap: default` remapping container root to unprivileged host UID |
| T-1-16 | Tampering | Temporal API accessible externally | mitigate | Temporal binds to 127.0.0.1:7233; UFW blocks external access |
| T-1-17 | Denial of Service | SSH brute force | accept | SSH allowed through UFW (necessary for admin); rate limiting via fail2ban is a Phase 7 hardening item (OSS-09) |
</threat_model>
