       **Types:**
       ```go
       type CommandRunner interface {
           Run(ctx context.Context, name string, args ...string) ([]byte, error)
       }

       type ExecCommandRunner struct{} // Real implementation using os/exec

       type Runner struct {
           cmd    CommandRunner
           logger zerolog.Logger
       }

       type RunOptions struct {
           Image     string
           Name      string
           Env       map[string]string
           Mounts    []string
           Network   string
           Memory    string
           CPUs      string
           PidsLimit string
           Remove    bool
           Labels    map[string]string
           Cmd       []string   // command to run inside container
       }

       type ContainerInfo struct {
           ID      string
           Name    string
           Status  string // "running", "exited", "created", "paused"
           Running bool
       }
       ```

       **Validation functions (critical for security):**
       - `validateContainerID(id string) error`: Must match `^[a-zA-Z0-9][a-zA-Z0-9_.-]*$`, max 128 chars, reject empty
       - `validateImageName(image string) error`: Must match `^[a-zA-Z0-9][a-zA-Z0-9_./-]*(:[a-zA-Z0-9_.-]+)?$`, max 256 chars, reject empty
       - `validateEnvVar(env string) error`: Must contain exactly one `=`, key must be `[A-Z_][A-Z0-9_]*`, value must NOT contain backticks, `$(`, or `\`` sequences
       - `validateMountPath(mount string) error`: Must be `host:container` format, no `..` path traversal, no shell metacharacters

       **Methods (all use `exec.CommandContext` via the `CommandRunner` interface):**

       - `func NewRunner(logger zerolog.Logger) *Runner` -- creates runner with real ExecCommandRunner
       - `func NewRunnerWithCmd(cmd CommandRunner, logger zerolog.Logger) *Runner` -- for testing

       - `func (r *Runner) Run(ctx context.Context, opts RunOptions) (string, error)`:
         Validate image, name, all env vars, all mounts. Build args: `["run", "-d"]`, append `--rm` if Remove, `--name` if Name, `--env` for each env var, `-v` for each mount, `--network` if set, `--memory` if set, `--cpus` if set, `--pids-limit` if set, `--label` for each label, then image, then Cmd. Execute `docker run`. Return trimmed container ID from stdout.

       - `func (r *Runner) Exec(ctx context.Context, containerID string, cmd []string) ([]byte, error)`:
         Validate containerID. Build args: `["exec", containerID]`, append cmd elements. Execute `docker exec`. Return stdout.

       - `func (r *Runner) Inspect(ctx context.Context, containerID string) (*ContainerInfo, error)`:
         Validate containerID. Execute `docker inspect --format '{{.Id}} {{.Name}} {{.State.Status}} {{.State.Running}}' <containerID>`. Parse output into ContainerInfo.

       - `func (r *Runner) Stop(ctx context.Context, containerID string) error`:
         Validate containerID. Execute `docker stop <containerID>`.

       - `func (r *Runner) Remove(ctx context.Context, containerID string) error`:
         Validate containerID. Execute `docker rm <containerID>`.

       - Port MSV's existing methods: `Kill`, `Logs`, `LogsSince` (renamed/adapted as needed)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Go code -> docker CLI | Untrusted values (container names, image names, env vars) passed to os/exec; command injection is the primary risk |
| container ID from external source | Session lifecycle will pass user-associated container IDs; if tampered, could target another user's container |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-03 | Tampering | docker CLI args | mitigate | validateContainerID/validateImageName/validateEnvVar reject shell metacharacters; exec.CommandContext uses arg arrays (no shell expansion) |
| T-1-07 | Elevation of Privilege | docker exec with arbitrary commands | mitigate | Exec accepts []string (no shell interpolation); callers must not pass unvalidated user input as cmd elements; Phase 5 further constrains allowed commands |
| T-1-08 | Information Disclosure | docker inspect exposes container details | accept | Inspect is called server-side only; container info is not exposed to end users in Phase 1 |
| T-1-09 | Denial of Service | Unbound container creation | accept | Phase 1 runner has no rate limit; Phase 5 session lifecycle enforces 1-active-per-user via DB constraint |
</threat_model>
