       ```go
       // DockerClient abstracts the Docker Engine SDK client for testability.
       // In production, this is satisfied by *client.Client from moby/moby/client.
       // In tests, use a mock implementation.
       type DockerClient interface {
           ContainerCreate(ctx context.Context, config *container.Config, hostConfig *container.HostConfig, networkingConfig *network.NetworkingConfig, platform *specs.Platform, containerName string) (container.CreateResponse, error)
           ContainerStart(ctx context.Context, containerID string, options container.StartOptions) error
           ContainerStop(ctx context.Context, containerID string, options container.StopOptions) error
           ContainerRemove(ctx context.Context, containerID string, options container.RemoveOptions) error

           ContainerExecCreate(ctx context.Context, container string, config container.ExecOptions) (types.IDResponse, error)
           ContainerExecAttach(ctx context.Context, execID string, config container.ExecAttachOptions) (types.HijackedResponse, error)
           ContainerExecInspect(ctx context.Context, execID string) (container.ExecInspect, error)
       }
       ```

       **Types (keep MSV's validation pattern but new SDK-based runner):**
       ```go
       type Runner struct {
           client DockerClient
           logger zerolog.Logger
       }

       type RunOptions struct {
           Image     string
           Name      string
           Env       map[string]string
           Mounts    []string              // "host:container" format
           Network   string
           Memory    int64                 // bytes (e.g. 512*1024*1024 for 512MB)
           CPUs      int64                 // in units of 1e9 (nanoCPUs)
           PidsLimit int64
           Remove    bool                  // auto-remove on exit
           Labels    map[string]string
           Cmd       []string              // command to run inside container
       }

       type ContainerInfo struct {
           ID      string
           Name    string
           Status  string // "running", "exited", "created", "paused"
           Running bool
       }
       ```

       **Validation functions (ported from MSV, critical for security):**
       - `validateContainerID(id string) error`: Must match `^[a-zA-Z0-9][a-zA-Z0-9_.-]*$`, max 128 chars, reject empty
       - `validateImageName(image string) error`: Must match `^[a-zA-Z0-9][a-zA-Z0-9_./-]*(:[a-zA-Z0-9_.-]+)?$`, max 256 chars, reject empty
       - `validateEnvVar(env string) error`: Must contain `=`, key must be `[A-Z_][A-Z0-9_]*`, value must NOT contain backticks, `$(`, or `` ` `` sequences
       - `validateMountPath(mount string) error`: Must be `host:container` format, no `..` path traversal, no shell metacharacters

       **Constructor:**
       - `func NewRunner(logger zerolog.Logger) (*Runner, error)`:
         Creates a real Docker SDK client using `client.NewClientWithOpts(client.FromEnv, client.WithAPIVersionNegotiation())`.
         Returns error if client creation fails.
       - `func NewRunnerWithClient(cli DockerClient, logger zerolog.Logger) *Runner`:
         For testing -- accepts a mock DockerClient.

       **Methods (all use the Docker Engine SDK, NOT os/exec):**

       - `func (r *Runner) Run(ctx context.Context, opts RunOptions) (string, error)`:
         1. Validate image, name (if set), all env vars, all mounts
         2. Build `container.Config`: Image, Env (as []string "K=V"), Cmd, Labels
         3. Build `container.HostConfig`: Memory, NanoCPUs, PidsLimit, AutoRemove, Binds (mounts), NetworkMode
         4. Call `r.client.ContainerCreate(ctx, containerConfig, hostConfig, nil, nil, opts.Name)`
         5. Call `r.client.ContainerStart(ctx, resp.ID, container.StartOptions{})`
         6. Return container ID from create response

       - `func (r *Runner) Exec(ctx context.Context, containerID string, cmd []string) ([]byte, error)`:
         1. Validate containerID
         2. Call `r.client.ContainerExecCreate(ctx, containerID, container.ExecOptions{Cmd: cmd, AttachStdout: true, AttachStderr: true})`
         3. Call `r.client.ContainerExecAttach(ctx, execID, container.ExecAttachOptions{})`
         4. Read all output from the hijacked connection
         5. Call `r.client.ContainerExecInspect(ctx, execID)` to check exit code
         6. Return stdout bytes; if exit code != 0, return error with stderr

       - `func (r *Runner) Inspect(ctx context.Context, containerID string) (*ContainerInfo, error)`:
         1. Validate containerID
         2. Call `r.client.ContainerInspect(ctx, containerID)`
         3. Map to ContainerInfo: ID=json.ID, Name=json.Name, Status=json.State.Status, Running=json.State.Running

       - `func (r *Runner) Stop(ctx context.Context, containerID string) error`:
         1. Validate containerID
         2. Call `r.client.ContainerStop(ctx, containerID, container.StopOptions{})` (default 10s grace period)

       - `func (r *Runner) Remove(ctx context.Context, containerID string) error`:
         1. Validate containerID
         2. Call `r.client.ContainerRemove(ctx, containerID, container.RemoveOptions{Force: false})`

       - Port MSV's existing read-only methods adapted to SDK:
         - `Kill` -> calls `r.client.ContainerStop` (same as Stop, mirrors MSV naming)
         - `Logs` -> calls Docker SDK logs API if needed in future (stub for now, not critical path)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Go code -> Docker Engine SDK -> Docker daemon | Untrusted values (container names, image names, env vars) passed via SDK structs; SDK uses daemon API socket |
| container ID from external source | Session lifecycle will pass user-associated container IDs; if tampered, could target another user's container |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-03 | Tampering | Docker SDK args | mitigate | validateContainerID/validateImageName/validateEnvVar reject shell metacharacters and malicious patterns before passing to SDK; SDK uses structured API calls (not shell) which adds a second layer of safety |
| T-1-07 | Elevation of Privilege | docker exec with arbitrary commands | mitigate | Exec accepts []string (no shell interpolation); callers must not pass unvalidated user input as cmd elements; Phase 5 further constrains allowed commands |
| T-1-08 | Information Disclosure | docker inspect exposes container details | accept | Inspect is called server-side only; container info is not exposed to end users in Phase 1 |
| T-1-09 | Denial of Service | Unbound container creation | accept | Phase 1 runner has no rate limit; Phase 5 session lifecycle enforces 1-active-per-user via DB constraint |
</threat_model>
