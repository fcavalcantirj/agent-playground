# Phase 1 — UI Design Contract

> Visual and interaction contract for the mobile-first login-gated landing page and dashboard shell. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (via `npx shadcn init`) |
| Style | new-york |
| Base Color | zinc |
| Component library | Radix UI (via shadcn) |
| Icon library | Lucide React (`lucide-react`) |
| Font | Inter (variable, via `next/font/google`) |

**Rationale:** shadcn new-york matches the BasicPay muscle memory (D-01 mirrors MSV shape). Zinc base color chosen over neutral to give Agent Playground a distinct developer-tool identity -- darker, crisper than BasicPay's purple-accented neutral. Inter is the shadcn default, highly legible at small mobile sizes, and ships free via `next/font`.

**shadcn initialization command:**
```bash
cd web && npx shadcn@latest init
```
Select: new-york style, zinc base color, CSS variables enabled, `@/components/ui` alias.

**shadcn components needed for Phase 1:**
```bash
npx shadcn@latest add button card
```

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, tight label-to-input spacing |
| sm | 8px | Compact element spacing, button icon gap |
| md | 16px | Default element spacing, card padding (mobile) |
| lg | 24px | Section padding, card padding (desktop) |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page vertical margins, hero section spacing |
| 3xl | 64px | Top-level page padding on desktop only |

Exceptions: Touch targets use 44px minimum height/width per D-13 (not a spacing token -- a component constraint). This applies to all buttons, links, and interactive elements on mobile viewports.

---

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Default text, descriptions, empty state copy |
| Label | 14px | 500 (medium) | 1.4 | Button text, nav items, metadata, badges |
| Heading | 20px | 600 (semibold) | 1.3 | Page headings, card titles, section headers |
| Display | 28px | 700 (bold) | 1.2 | Hero headline on the landing page only |

**Mobile-first note:** Body at 16px prevents iOS zoom-on-focus behavior. Do not use body text below 16px on mobile. Label at 14px is acceptable only for non-input, non-editable text.

## Color

Agent Playground uses shadcn's zinc base with a custom emerald accent. Dark mode is the default (developer-tool convention). Light mode is supported but secondary.

### Dark Mode (default)

| Role | Value | CSS Variable | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `hsl(240 10% 4%)` | `--background` | Page background, surfaces |
| Secondary (30%) | `hsl(240 6% 10%)` | `--card` | Cards, dashboard shell, nav bar |
| Accent (10%) | `hsl(160 84% 39%)` | `--primary` | See reserved list below |
| Destructive | `hsl(0 84% 60%)` | `--destructive` | Destructive actions only (none in Phase 1) |

### Light Mode

| Role | Value | CSS Variable | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `hsl(0 0% 100%)` | `--background` | Page background |
| Secondary (30%) | `hsl(240 6% 97%)` | `--card` | Cards, nav |
| Accent (10%) | `hsl(160 84% 39%)` | `--primary` | Same reserved list |
| Destructive | `hsl(0 84% 60%)` | `--destructive` | Destructive actions only |

**Accent reserved for (explicit list -- never "all interactive elements"):**
1. Primary CTA button ("Sign in" / "Dev Login")
2. Active navigation indicator (bottom tab highlight when dashboard is built out)
3. Session status badge in "running" state (future phases, pattern set now)
4. The "play" icon or "launch" verb in any call-to-action

**Accent is NOT used for:** links (use foreground with underline), secondary buttons (use outline variant), borders, dividers, or decorative elements.

**Color identity rationale:** Emerald green (hsl 160) signals "go / launch / active" -- aligns with the product's core action (launching agent sessions). Differentiates from BasicPay's purple (#5067EB / oklch 265 hue).

## Responsive Breakpoints

Mobile-first (D-12). Styles are written for mobile, then enhanced upward.

| Breakpoint | Width | Layout Change |
|------------|-------|---------------|
| Base (mobile) | < 640px | Single column, bottom navigation, full-bleed cards |
| sm | >= 640px | Slight padding increase, cards get border-radius |
| md | >= 768px | Two-column possible, side margins appear |
| lg | >= 1024px | Max-width container (960px), centered layout |
| xl | >= 1280px | No additional changes in Phase 1 |

**Max content width:** 960px at lg+, centered with auto margins. Phase 1 has no sidebar.

---

## Navigation Pattern

**Mobile (< 768px):** No persistent navigation in Phase 1. The landing page is a single-screen flow. When the dashboard shell expands in Phase 5, use a bottom tab bar (tabs for agents, matching D-16 "agents appear as tabs for quick-switch chat").

**Desktop (>= 768px):** Minimal top bar with: product name (left), user avatar + sign-out (right). No sidebar in Phase 1.

**Phase 1 navigation structure:**
- `/` -- Landing page (unauthenticated: sign-in prompt; authenticated: dashboard shell)
- No other routes in Phase 1

## Page Contracts

### Screen 1: Unauthenticated Landing (sign-in prompt)

**Layout:** Vertically centered on mobile. Single column, max-width 400px on desktop.

| Element | Spec |
|---------|------|
| Logo/wordmark | "Agent Playground" in Display size (28px/700), centered |
| Tagline | "Any agent. Any model. One click." in Body size (16px/400), centered, muted foreground |
| Mission line | "The easiest way to deploy any coding agent" in Label size (14px/500), centered, muted foreground |
| Dev Login button | Full-width on mobile (min-height 44px), accent background, white text, Label weight. Text: "Dev Login". Rounded corners (--radius). |
| Dev mode badge | Below button: "Development mode" in Label size, muted, with a terminal icon (Lucide `Terminal`) |
| Spacing | 16px gap between tagline and mission. 32px gap between mission and button. 8px gap between button and dev-mode badge. |

**Behavior:** Clicking "Dev Login" sends `POST /api/dev/login` with no body. On 200, redirect to `/` (which now renders the authenticated dashboard). On error, show inline error below button.

### Screen 2: Authenticated Dashboard Shell

**Layout:** Full viewport height. Top bar + main content area.

| Element | Spec |
|---------|------|
| Top bar | Height 56px (44px content + padding). Background: card color. Left: "Agent Playground" in Heading size (20px/600). Right: user display name in Label size + avatar circle (32px) + sign-out icon button (Lucide `LogOut`, 44px touch target). |
| Main content | Centered vertically and horizontally in remaining space. |
| Empty state icon | Lucide `Bot` icon, 48px, muted foreground |
| Empty state heading | "No agents yet" in Heading size (20px/600) |
| Empty state body | "Your coding agents will appear here. Agent setup arrives in a future update." in Body size (16px/400), muted foreground, max-width 280px, centered |
| Footer hint | "Phase 1 -- Foundation shell" in Label size (14px/500), muted, bottom of viewport with 16px padding. Only visible in dev mode. |

**Behavior:** Sign-out button sends `POST /api/dev/logout` (or `DELETE /api/dev/session`), clears cookie, redirects to `/` unauthenticated state.

## Interaction States

| State | Visual Treatment |
|-------|-----------------|
| Button hover | Slight brightness increase (opacity 0.9 on accent bg) |
| Button active/pressed | Scale down to 0.98, opacity 0.8 |
| Button loading | Lucide `Loader2` spinner replacing text, disabled state, opacity 0.7 |
| Button disabled | Opacity 0.5, cursor not-allowed |
| Focus visible | 2px ring using `--ring` color, 2px offset. All interactive elements. |
| Error state | Red text (destructive foreground) below the element, 8px gap |
| Skeleton loading | shadcn `Skeleton` shimmer on card areas during auth check |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Touch targets | 44px minimum height AND width for all interactive elements (D-13) |
| Focus management | Visible focus ring on all interactive elements. Tab order: logo (skip) -> dev login button. Dashboard: logo -> sign out. |
| Color contrast | WCAG AA minimum (4.5:1 for body text, 3:1 for large text). Emerald accent on dark bg passes -- verify during implementation. |
| Reduced motion | Respect `prefers-reduced-motion`: disable scale transitions, use instant state changes. |
| Screen reader | Button labels are self-describing. Empty state uses `role="status"`. Sign-out icon button has `aria-label="Sign out"`. |

## Dark Mode Strategy

Dark mode is the default. Implement via `class` strategy (not `media`) so users can override. In Phase 1, ship dark-only. Light mode CSS variables are declared but the toggle is not exposed until a later phase.

**Implementation:** Set `<html class="dark">` in the root layout. shadcn's CSS variable system handles the rest.
