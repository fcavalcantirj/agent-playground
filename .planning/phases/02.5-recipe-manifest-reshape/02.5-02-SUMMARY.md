# Phase 02.5 Plan 02: TemplateRegistry Renderer Summary

**One-liner:** Hardened filesystem template registry with closed 5-function FuncMap, three-layer path validation, goroutine-bounded 5s/64 KiB execution caps, and mtime-keyed parse cache â€” implements REC-07 and closes threat T-02.5-03 (SSTI) by construction.

## What Shipped

### `api/internal/recipes/template.go` (263 lines)

- `TemplateRegistry` struct with `root string`, `sync.RWMutex`, `map[string]cachedTmpl` cache, and a test-only `parseCount int`.
- `NewTemplateRegistry(root string) *TemplateRegistry` â€” plain constructor; root points at `agents/`.
- `Render(ctx, recipeID, name, data) (string, error)` â€” the single public entry point:
  1. Append `.tmpl`, validate with `templateNameRE = ^[a-z0-9][a-z0-9._-]*\.tmpl$`.
  2. `os.Lstat` â€” reject if `info.Mode()&os.ModeSymlink != 0`.
  3. `filepath.Abs(dir)` + `filepath.Abs(path)` + `strings.HasPrefix(absPath, absDir+sep)` â€” reject resolved-path escape.
  4. Cache lookup by `absPath`; invalidate on `!mtime.Equal(info.ModTime())`; re-parse with `template.New(filename).Funcs(safeFuncs).Option("missingkey=error").Parse(...)`.
  5. Execute on a goroutine, write to `*limitedWriter{max: 65536}`, `select` on `context.WithTimeout(ctx, 5*time.Second)` vs result channel.
- Sentinel errors: `ErrTemplatePath`, `ErrTemplateSize`, `ErrTemplateTimeout`.
- `ParseCountForTest()` â€” the only test-only API surface.
- `limitedWriter` â€” cumulative cap; first Write that would cross the cap returns `(0, ErrTemplateSize)` without issuing a partial write.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Regex allowlist was inconsistent with fixture filename**

- **Found during:** TDD GREEN â€” `TestRender_Valid` failed with `ErrTemplatePath: "security.yml.tmpl"`.
- **Issue:** The plan specifies `templateNameRE = ^[a-z0-9][a-z0-9_-]*\.tmpl$` AND the fixture `security.yml.tmpl` in the same plan. Because the char class excludes `.`, the fixture filename is rejected by the very regex the plan tells us to use. The `template` field convention (`template: security.yml` â†’ disk `security.yml.tmpl`) is documented in the plan's `<template_extension_convention>` block, so the fixture is canonical and the regex is the buggy artifact.
- **Fix:** Widened the intermediate char class to include `.`: `^[a-z0-9][a-z0-9._-]*\.tmpl$`. The leading-char class still excludes `.`, so `..` and `.hidden` are still rejected by the first layer. Documented in template.go with a multi-line comment.
- **Files modified:** `api/internal/recipes/template.go`.
- **Commit:** `7d66fcc` (the single GREEN commit; the fix was applied before the commit landed).
- **Defense in depth intact:** symlink reject (layer 2) and absolute-prefix check (layer 3) unchanged; the widened regex only affects layer 1.
