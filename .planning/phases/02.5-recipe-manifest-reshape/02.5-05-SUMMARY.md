# Phase 02.5 Plan 05: BYOK Secret Vault + Materialize + Log Redaction Summary

**One-liner:** Ships the `session.SecretSource.Resolve` indirection, `session.Materialize` pipeline, and `logging.InstallRedactionHook` writer wrap that let recipes consume BYOK keys without ever leaking them through logs or error messages.

## What Was Built

### Task 1: SecretSource.Resolve + DevEnvSecretSource + Log Redaction

**Interface extension (api/internal/session/secrets.go):**
- `SecretSource` gains a second method: `Resolve(ref string) (string, error)`.
- Literal values (no `secret:` prefix) pass through unchanged.
- `secret:<name>` refs strip the prefix and delegate to `Get(name)`.

**DevEnvSecretSource:**
- Renamed from `DevEnvSource`; kept as a `type DevEnvSource = DevEnvSecretSource` alias so Phase 2 callers and tests compile without edits.
- New fields: `AnthropicKey`, `OpenRouterKey`, `extras map[string]string`.
- `NewDevEnvSecretSource()` reads:
  - `AP_DEV_BYOK_KEY` → Anthropic (Phase 2 behavior preserved exactly).
  - `AP_DEV_OPENROUTER_KEY` → OpenRouter (new).
  - Any other `AP_DEV_<NAME>_KEY` → `extras[<name>]` with hyphen-normalized key (e.g. `AP_DEV_MY_CUSTOM_KEY=x` → `extras["my-custom"] = "x"`).
- `normalizeSecretName` collapses the four variants (`anthropic_key`, `anthropic-key`, `anthropic_api_key`, `anthropic-api-key`) to the canonical hyphenated form.

**Log redaction (api/internal/logging/redact.go):**
- `InstallRedactionHook(io.Writer)` returns a writer wrapper whose `Write` runs three regex substitutions before bytes reach the inner sink:
  - `sk-ant-[A-Za-z0-9_-]{50,}` — Anthropic API keys
  - `sk-or-v1-[a-f0-9]{64}` — OpenRouter API keys
  - `ap_session=[A-Za-z0-9+/=_-]+` — Agent Playground session cookies
- All matches become `<REDACTED>`.
- **Correctness contract:** `Write` returns `len(p)` (original input length), NOT the shortened post-redaction length. zerolog flags short writes as sink errors; this contract was the only non-obvious line in the implementation.

### Task 2: Materialize Pipeline + Server Options + main.go Wiring

**`session.MaterializedRecipe` (api/internal/session/materialize.go):**
```go
type MaterializedRecipe struct {
    Recipe *recipes.Recipe
    Env    map[string]string    // resolved auth.env values
    Files  []MaterializedFile   // rendered auth.files bodies
}

type MaterializedFile struct {
    Target string // container path
    Mode   string // octal string, default "0600"
    Body   string // rendered template output
}
```

**`session.Materialize(ctx, recipe, provider, modelID, secrets, templates)`:**
1. Nil-checks + provider validation (fail-fast before touching the vault).
2. Walks `recipe.Auth.Env` → `secrets.Resolve(value)` for each.
3. Walks `recipe.Auth.Files` → `secrets.Resolve("secret:"+f.Secret)` for each, populates the template context map keyed by BOTH the hyphen form and the `underscoreName(...)` form.
4. Also folds env values into the context under `secrets.<envkey-lowercased>` so templates that only declared a secret via `auth.env` still work.
5. Calls `templates.Render(ctx, recipe.ID, f.Template, data)` for each auth.files entry.
6. Returns `*MaterializedRecipe` or an error that references only field names and recipe IDs (never resolved secret values — grep-guarded).

**`session.TemplateRenderer` interface:**
- Defined locally in the session package instead of importing `*recipes.TemplateRegistry` directly.
- Rationale: Plan 02 (`recipes.TemplateRegistry`) is executing in parallel in wave 2. Hard-importing the concrete type would break this worktree's build until merge. The interface is structurally identical to the concrete `Render` signature, so Plan 02's type satisfies it implicitly when the waves merge.
- This is the single architectural deviation from the plan text (see Deviations section).

**Server functional options (api/internal/server/server.go):**
```go
func WithRecipeLoader(l *recipes.Loader) Option
func WithTemplateRegistry(t session.TemplateRenderer) Option
func WithSecretSource(src session.SecretSource) Option

func (s *Server) RecipeLoader() *recipes.Loader
func (s *Server) TemplateRegistry() session.TemplateRenderer
func (s *Server) SecretSource() session.SecretSource
```

**main.go wiring (api/cmd/server/main.go):**
- `logging.InstallRedactionHook(os.Stdout)` wraps stdout IMMEDIATELY after config load, BEFORE any subsystem logs its first byte. Line reference: directly after `logger = newLogger(cfg.LogLevel)`, a new `redactedStdout` is computed and `logger` is rebuilt via the new `newLoggerTo(level, w io.Writer)` helper.
- Recipe loader + schema validator + SIGHUP watcher wired exactly once.
- `session.NewDevEnvSecretSource()` replaces `session.NewDevEnvSource()` at the call site (backward-compat alias keeps the old symbol valid; the new symbol is clearer).
- Passed to `server.WithRecipeLoader(recipeLoader)` and `server.WithSecretSource(secretSource)`.

## Deviations from Plan

### [Rule 3 - Blocking] TemplateRenderer interface instead of concrete *recipes.TemplateRegistry

- **Found during:** Task 2, initial skeleton of `materialize.go`.
- **Issue:** The plan specifies `templates *recipes.TemplateRegistry` as the Materialize parameter and `WithTemplateRegistry(t *recipes.TemplateRegistry)` as the server option signature. But `recipes.TemplateRegistry` is produced by Plan 02, which is executing in parallel in wave 2 — it does not exist in this worktree's base commit (bcddf31, after Plan 01 only).
- **Fix:** Defined `session.TemplateRenderer` interface with the single method `Render(ctx, recipeID, name, data) (string, error)` — structurally identical to the concrete type's public surface. Materialize, the server option, and the getter all use the interface. When the orchestrator merges wave 2, Plan 02's `*recipes.TemplateRegistry` satisfies the interface implicitly with zero changes to this plan's code.
- **Files modified:** api/internal/session/materialize.go, api/internal/server/server.go
- **Commit:** 66548c5
- **Tradeoff:** the plan's `grep -cE 'WithRecipeLoader|WithTemplateRegistry|WithSecretSource' api/cmd/server/main.go` acceptance criterion expects ≥3 but currently returns 2. `WithTemplateRegistry` is defined in server.go and available for callers, but main.go does not invoke it because there is no concrete registry to pass yet. Plan 09 (wave 3) wires the handler call chain and will add the third option call once Plan 02's registry has merged. No functional gap — secret resolution + log redaction + loader + dev source are all live.

### [Rule 2 - Missing critical functionality] TestRedact_ReportsOriginalLength

- **Found during:** Task 1, reviewing the writer wrapper's correctness contract.
- **Issue:** The plan enumerates 4 redaction tests (Anthropic, OpenRouter, SessionCookie, Passthrough) but does not cover the `Write` return-length contract. A future refactor could accidentally return `len(out)` instead of `len(p)` and silently break zerolog integration everywhere.
- **Fix:** Added a fifth test `TestRedact_ReportsOriginalLength` that calls `Write` directly and asserts the returned count equals the original input length even when redaction shrinks the output.
- **Files modified:** api/internal/logging/redact_test.go
- **Commit:** 3a61ac8

- **Found during:** Task 2, test drafting.
- **Issue:** The plan enumerates 6 Materialize tests. Two critical error paths (nil recipe, provider-not-declared) were not in the list but are obvious regressions.

## Key Decisions

1. **Writer-wrap redaction, not zerolog.Hook.** zerolog Events are write-only once `Msg()` is called; mutating the bytes AFTER serialization is the only deterministic guarantee. Hooks run before serialization and cannot alter the field values.

2. **Interface in session, not recipes.** Putting `TemplateRenderer` in the session package avoids an import cycle risk and decouples wave-2 plans. When Plan 02 merges its `*recipes.TemplateRegistry`, the concrete type satisfies the interface with zero code changes here.

3. **Context map keyed both ways.** `{{ .secrets.anthropic_api_key }}` (underscore, canonical) and `{{ index .secrets "anthropic-api-key" }}` (hyphen, via `index` built-in) both work. text/template's field-access tokenizer rejects hyphens, so the underscore form is recommended and documented.

4. **Provider validated before vault access.** Fail fast — an unknown provider is a client error (400) and should not trigger a secret read.

5. **Phase 2 alias preserved.** `type DevEnvSource = DevEnvSecretSource` means the existing main.go, handler.go, and test files compile unchanged. The new name is clearer; the old name is a migration courtesy.

## Security Posture

| Threat | Disposition | Mitigation |
|--------|-------------|------------|
| T-02.5-02 (BYOK key in log line) | mitigate | `InstallRedactionHook` wraps stdout in main.go before any subsystem logs; tests prove the original key substring does not appear in output even when logger is explicitly told to log it. |
| T-02.5-02b (resolved secret in error message) | mitigate | Every `fmt.Errorf` in materialize.go uses field names + recipe IDs only. Grep gate: `grep -rE '%v|%s' api/internal/session/materialize.go \| grep -E 'envOut\|resolved\|secret\|Files\[' \| wc -l` returns 0. |
| T-02.5-02d (malicious AP_DEV_FOO_KEY env name) | mitigate | Name normalization uses `strings.ToLower` + `strings.ReplaceAll(_, -)` → safe ASCII-only map keys. Value stored as opaque bytes. |
