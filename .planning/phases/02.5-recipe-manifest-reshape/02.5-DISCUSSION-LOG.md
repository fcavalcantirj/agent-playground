**User's choice (verbatim):** *"dude, this is very specific. are we using docker? lovely? each agent must probably have a pattern on its docs, right? some we might override, some dont. but, we also agreed in a folder, on the repo, with the 'template' for the agents, so every combination possible on the interface, we probably have a template on the open github. make sense?"*

**Recovery:** summarized Phase 2's goal ("API-driven start without Telegram is the hypothesis proof, just curl → real model reply") with the 7 concrete deliverables Phase 2 shipped.

**Third attempt:** re-asked with both the chat_io context and Phase 2's goal grounded.

**User response (verbatim):** *"HTTP API call, run inside a sandboxed Docker container, talk to a real LLM using a user-supplied key, and respond to a user message — without Telegram, without a browser UI or simple api call, WITH Temporal, just curl against the Go API works, BROWSER UI WE CAN DEFER."*

**Interpretation:** user re-affirmed Phase 2's scope, added an ambiguous "WITH Temporal" clause, and re-confirmed no browser UI. Did NOT answer the chat_io question directly; the "focus on what matters" signal implied cut-to-4. Deferred the chat_io lock until after disambiguating the Temporal clause.

**Final lock (D-10):** Cut to 4 (fifo, exec_per_message, one_shot_task, http_gateway). Schema version v0.1.0. Closed enum. No dead code. When a future agent forces grpc_uds / json_rpc_stdio / terminal_only, schema bumps to v0.2.
