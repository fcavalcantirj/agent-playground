## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Aider upstream at pinned SHA → recipe install | We trust Aider-AI/aider at the pinned commit. Install happens at session-start time via pypi. |
| pypi install → agent container | uv pip install downloads from pypi. Trust = pypi infrastructure + aider package maintainers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02.5-01 | Tampering | Malicious aider-chat package replaces upstream | accept | pypi supply chain trust — same as every other Python project. Pinned version via `version: ">=0.60"` + pinned source_sha for source attribution. Future (Phase 8+) will consider pip install with `--require-hashes`. v0.1 accepts pypi tag. |
| T-02.5-01a | Tampering | Recipe YAML smuggles shell metacharacters into lifecycle hook | mitigate | Plan 01 schema validates every hook as a `hook` $def (string or array-of-strings or array-of-arrays). This recipe uses array-of-strings for postCreateCommand (`[uv, pip, install, --system, --no-cache, aider-chat]`), so each element is a literal argv value. No sh -c, no shell interpolation. |
| T-02.5-02 | Information Disclosure | ANTHROPIC_API_KEY leaks via aider stdout (e.g. error message) | mitigate | Plan 05's redacting writer wraps stdout for the API server. aider's own stdout goes through the ExecBridge which reads via docker exec — the ExecBridge output is logged via the server logger which is redacted. If aider itself echoes the key back in an error message, the regex scrubber catches it before logging. |
| T-02.5-04 | Elevation of Privilege | `aider --yes-always --no-auto-commits --no-dirty-commits` grants autonomy that could write outside workspace | mitigate | Recipe runs under Phase 2 sandbox: cap_drop=ALL, read_only_rootfs=true, agent user post-gosu. Aider can only write to tmpfs `/home/agent/.aider` + `/tmp` + `/run`. No host bind mounts. No network egress control in 02.5 (Phase 7.5 adds `ap-net` allowlist). The `--yes-always` flag removes interactive prompts — that's a usability gain for the smoke test, not a security regression, because the sandbox posture is unchanged. |
