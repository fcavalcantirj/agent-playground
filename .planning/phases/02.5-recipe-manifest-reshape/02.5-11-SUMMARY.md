## Decisions Made

See key-decisions in the frontmatter. Two worth highlighting inline:

1. **Synthetic fallback over a real agent** (D-50d over D-50b alternatives). The orchestrator note recommended picking "a real third-party agent that exercises the same substrate as aider" with a small python/uv footprint. null-echo fits that description but is even cleaner — it has no upstream CLI to depend on, so its failure modes can only be architectural (substrate rejects it) or wiring (cmd_template wrong), never "the vendor shipped a breaking change". This isolates the Gate B variable perfectly.

2. **Python 3.13 over 3.12** — aider had to fall back to 3.12 because of a numpy cp313 wheel problem (see aider's recipe.yaml header). null-echo has no such constraint and explicitly targets 3.13, which proves the primary runtime base image (`ap-runtime-python:v0.1.0-3.13`) absorbs a recipe as cleanly as the 3.12 fallback does. This widens the Gate B proof from "substrate + 3.12 overlay" to "substrate + both python overlays".
