Use the **short-circuit-via-raise pattern** — DO NOT build a multi-state fake subprocess. The fake subprocess only intercepts the FIRST `docker run -d` call and raises a sentinel exception carrying the captured cmd + env-file content; everything before that point (env-file write — note: that's a Path.write_text, not subprocess) executes normally. The test then catches the sentinel and asserts.

   **Concrete pattern (verbatim — to be embedded in `api_server/tests/test_run_recipe_telegram_invariant.py`):**
   ```python
   class _SnapshotCaptured(Exception):
       def __init__(self, cmd, env_file_content):
           self.cmd = cmd
           self.env_file_content = env_file_content

   def _fake_subprocess_run(cmd, *args, **kwargs):
       # Only intercept the FIRST `docker run -d` call — let everything before it
       # (e.g., env-file write — that's Path.write_text, not subprocess) execute normally.
       if cmd[:3] == ["docker", "run", "-d"]:
           # Read the env-file (must be written before docker run)
           env_file_path = next(
               cmd[i+1] for i, c in enumerate(cmd) if c == "--env-file"
           )
           env_content = Path(env_file_path).read_text()
           raise _SnapshotCaptured(list(cmd), env_content)
       # Any other subprocess call (shouldn't happen pre-run) — fail loud
       raise AssertionError(f"unexpected subprocess.run call: {cmd[:3]}")
   ```
