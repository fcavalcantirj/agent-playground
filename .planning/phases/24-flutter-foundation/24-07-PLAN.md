## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Flutter app → backend (dio HTTP) | Cleartext traffic crosses iOS ATS / Android NetworkSecurityConfig gates |
| OS browser → app (deep link) | OAuth redirect crosses solvrlabs:// custom URL scheme |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-07-01 | Information Disclosure | iOS ATS exemption | mitigate | `NSExceptionDomains.localhost` is the ONLY allowlisted domain; `NSAllowsLocalNetworking=true` covers RFC1918 ranges. Production HTTPS still enforced for everything else. |
| T-24-07-02 | Information Disclosure | Android cleartext config | mitigate | Cleartext config lives at `src/debug/res/xml/` and is merged ONLY into debug builds via `src/debug/AndroidManifest.xml`. Release builds retain default `cleartextTrafficPermitted=false`. |
| T-24-07-03 | Spoofing | Custom URL scheme `solvrlabs://` | accept | Custom URL schemes are inherently squatable on Android. D-15 captures this — Universal Links are deferred to a future hardening once a verified HTTPS domain exists. PKCE in flutter_appauth (Phase 25) defends against malicious app intercepting the redirect. |
| T-24-07-04 | Tampering | applicationId mismatch | mitigate | applicationId in build.gradle.kts MUST match iOS bundle id (D-02). Single canonical value `com.solvrlabs.agentplayground` enforced by grep gate; no `agent_playground` underscore variant remains. |
</threat_model>
